Phase 3:
Evaluation of overall residual risk acceptability

During this phase, the MDS AI model should be assessed for overall residual risk acceptability. This framework recommends conducting Penetration testing (pentesting) to evaluate the overall residual risk acceptability of the AI model.

Penetration testing (pentesting) involves conducting simulated attacks on the AI model to identify and exploit potential vulnerabilities, ensuring the security and robustness of the AI model.

Steps for conducting Pentesting

Step 01: Preparation

Defining the scope of testing includes identifying the following:
  • A list of assets to be considered for testing - Possible list of assets
  • A list of threats and vulnerabilities for which the controls are implemented - Possible list of threats and vulnerabilities
  • Objectives of the testing - Eg: Identify adversarial vulnerabilities, test for data/model leakage, check security of APIs etc...

Information gathering includes the following:

  • The type of model (e.g., classifier, regressor, image model, etc...), framework used (e.g., TensorFlow, PyTorch, Scikit-learn, HuggingFace), and tasks (e.g., classification, regression).
  • Input/Output Structure - What kind of data is accepted (e.g., text, image, JSON)?

Step 02: Testing

This step includes simulating data security attacks to test the effectiveness of the implemented security controls. The attacks may include the following: (A comprehensive list of possible attacks is available Here!
  • Injecting adversarial inputs - Create minimally modified inputs that confuse the model using tools such as: CleverHans, Foolbox, ART (Adversarial Robustness Toolbox).
  • Attempting data poisoning - Inject malicious samples to observe whether they degrade model performance, using tools such as: IBM ART, TrojAI.
  • Exploiting insecure APIs - Attempt to bypass authentication and authroisation, data encryption in transit and storage etc...using tools such as: OWASP ZAP, Burp suite.
Subsequently the AI model’s response to these attacks should be assessed to identify any weaknesses that could be exploited. For example, the following metrics can be used to assess the repsonse to simulation attacks:
  • Accuracy/FI Score (before vs. after attack)
  • Precision/Recall
  • Confidence Scores
  • Attack Success Rate (ASR)
  • Robustness Score
  • Perbutaion Sensitivity

Step 03: Reporting and Follow up

Upon completion of the simulated attacks, a detailed penetration test report of the newly identified vulnerabilities should be prepared for analysis.

After conducting AI pentesting, the following steps needs to be assess overall residual risk acceptability:

Steps for Conducting Evaluation of the Overall Residual Risk Acceptability based on the Penetration Test Report

Upon completion of Pentesting, the test results report should be reviewed to evaluate the overall residual security risk acceptability. Passing Pentesting does not indicate that the MDS AI model is invulnerable; it only means that the model resisted the forms of simulated attacks. If the Pentesting is successful, the developer can decide that the MDS AI model is ready to be deployed for real-world use. If the test fails, the following tasks should be conducted:
  • Check whether the vulnerability is new or an existing vulnerability which is already listed in the framework.
  • If the vulnerability is an existing one, review the list of risk controls provided in the framework to check whether the control is inappropriate or improperly implemented without following the implementation details provided. After identifying the reason for the failure, perform suitable measures to address the identified reason for failure.
  • If the vulnerability is a new one, check whether the framework provides appropriate risk controls to mitigate it. If yes, implement the risk control by following the implementation details provided in the framework. If no, identify appropiate risk controls and implementation details from external sources such as NIST 800-53, ISO 27001/02, IEC 27799, OWASP, Plot4ai, other relevant technical reports, scientific papers, websites, online databases etc…
  • Update the framework by documenting the newly identified vulnerabilities, controls and control implementation details.
  • After implementating the risk controls, conduct AI Pentesting again to verify whether the implemented control successfully mitigates the vulnerability.
  • If the test is succsessful, the developer can mark the AI model as suitable for deployment in the real-world.
  • If the test is unsuccessful, further risk controls need to be implemented.
